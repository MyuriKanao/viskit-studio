# Step 2 — Selling-point cards

Distill 3-5 selling points, each carrying:

- Audience → pain point → solution → benefit → trust evidence → priority → applicable module

Every point must rest on **verifiable facts** (patent number / lab report / certification / craft).
**Gate rule**: selling points must pass compliance review (no absolutes, no effect implications, no medical claims).

In headless API mode the user-confirmation gate has already been resolved by the caller — selling points arrive structurally validated.
